# Linux Fundamentals

## Why Learn Linux First?

Mastering Linux isn't optional for DevOps—it's a requirement. Here's what happens if you skip the fundamentals:

| If you skip this... | What happens in the real world |
|---------------------|--------------------------------|
| You don't know Linux | You can't debug why your container won't start |
| You don't know file permissions | Your app can't write logs |
| You don't know processes | You can't find the "zombie" process eating CPU |
| You don't know networking | You can't figure out why two containers can't talk |

---

> **Linux is the operating system that runs 90% of the world's servers. You MUST be comfortable in the terminal.**

### Key Topics:
- Navigating the filesystem (`cd`, `ls`, `pwd`, `tree`)
- File permissions and ownership (`chmod`, `chown`)
- Process management (`ps`, `top`, `htop`, `kill`)
- Networking basics (`ping`, `curl`, `netstat`, `ss`)
- Package management (`apt`, `yum`, `dnf`)
- Text editors (`vim`, `nano`)
- Logs and monitoring (`tail`, `journalctl`)
